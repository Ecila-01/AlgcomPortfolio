Graphs

A graph consists of two sets: Vertices and Edges. vertices are the point in graph while edges are a pair of vertices that are connected. Graph can either be Directed - One that has a specific direction, Indirected - One that has no specific direction and can go either way and Weighted, one that has a particular cost or weight for being connected.

In order to plot out a graph, we can use adjacency matrix that is a table consisting of columns and row of the point and we can plot it by 1s and 0s depending of the point's connections

Graph Traversal

There are multiple ways for traversing through graphs. In this course, we learned about Breadth First Search (BFS) and Depth First Search (DFS). Depth First Search (DFS) starts at a certain point and explores as far as possible along each branch before backtracking. Breadth First Search (BFS) on the other hand starts also at a certain point but explores all nodes at the present depth prior to moving on to the nodes at the next depth level.


Dijkstra’s Algorithm

On weighted graph, there are centain cost for traversing through one point to another. Dijkstra’s Algorithm is done to be able to traverse through a graph with the shortest possible route. It uses the adjacency matrix to properly plot the graph into a table to find the shortest path

Asymptotic Analysis

Asymptotic analysis refers to computing the running time of any operation in mathematical units of computation. For example, the running time of one operation is computed as f(n) and may be for another operation it is computed as g(n2). It provides a simplified model of the running time or other resource needs of an algorithm to help you understand the behavior of your algorithms


Calculations:

n f(n) g(n) c = f(n) / g(n) c * g(n)
                             
                             
                             

Big-Oh (Big-O) Notation: f(n) = O(g(n)) iff f(n) ≤ c * g(n) where n ≥ n0 , c > 0, n0 ≥ 1

n f(n) g(n) g(n) / f(n) c = 1/(g(n) / f(n)) c * g(n)
                                   
                                   
                                   

Big-Omega (Big-Ω) Notation: f(n) = Ω (g(n)) iff f(n) ≥ c * g(n) where n ≥ n0, c > 0, n0 ≥ 1

n f(n) g(n) g(n) / f(n) c1 = 1/(g(n) / f(n)) c1 * g(n) c2 = f(n)/g(n) c2 * g(n)
                                               
                                               
                                               

Big-Theta (Big-Θ) Notation: f(n) = Θ(g(n)) iff c1 * g(n) ≤ f(n) ≤ c2 * g(n) where n ≥ n0, c1 and c2 > 0, n0 ≥ 1
